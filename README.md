# AVR-Based Smart Home Automation and Security System

## Overview

This repository contains an AVR-based smart home automation and security system originally developed as an undergraduate capstone project in 2009.

The project was implemented using Atmel AVR microcontrollers and the C programming language within the CodeVisionAVR development environment. The system integrates environmental monitoring, relay scheduling, security protection, serial communication, and user interface management into a standalone embedded platform.

This repository is published primarily for archival, educational, and historical engineering purposes.

---

# Features

* AVR-based embedded architecture
* Dual-microcontroller distributed system
* USART serial communication
* Relay scheduling and automation
* DS1307 Real-Time Clock integration
* Temperature monitoring and cooling control
* Indoor and outdoor light detection
* MQ-5 gas leakage monitoring
* Laser-based intrusion detection
* Password-protected security mode
* LCD-based user interface
* Keypad configuration system
* Persian (Shamsi) calendar conversion

---

# System Architecture

The system consists of two independent hardware boards:

## 1. Transmitter / Controller Unit

The transmitter board acts as the main processing controller responsible for:

* sensor acquisition
* automation logic
* scheduling
* RTC management
* user interface handling
* security monitoring
* communication packet generation

## 2. Receiver / Actuator Unit

The receiver board performs:

* serial packet reception
* checksum validation
* relay control
* alarm activation
* output switching

Communication between the two boards is implemented using USART serial communication at 9600 baud.

# Communication Protocol

The system uses a custom serial communication protocol.

## Packet Structure

| Byte | Description       |
| ---- | ----------------- |
| 0    | Start Byte (0xFF) |
| 1-12 | Command Data      |
| 13   | Checksum          |

The checksum is generated by summing all command bytes before transmission.

# Historical Context

This project was originally developed in 2009 as a Bachelor of Science final-year embedded systems project. At the time, low-cost AVR microcontrollers were widely used for standalone automation systems before the widespread adoption of IoT-based architectures.

The implementation reflects the embedded design methodologies and hardware constraints of that period, including:

* polling-based execution
* simple checksum validation
* direct register-level peripheral interfacing
* monolithic embedded software architecture

---

# Limitations

The original implementation contains several limitations typical of early embedded systems projects:

* Polling-based processing instead of interrupts
* Simple checksum error detection
* No wireless communication
* Password storage in RAM
* Limited software modularity
* No RTOS support

These limitations are intentionally preserved as part of the historical implementation.
